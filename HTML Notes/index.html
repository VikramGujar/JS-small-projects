Introduction to Web

The web is a vast platform that includes various types of software applications. Here are the main categories:

Types of Software Applications:

  1. Desktop Application: Installed on our machine. Examples include Microsoft Excel, PowerPoint, etc.
  2. Web Application: Internet-based applications. Examples include Amazon, Facebook, etc.
  3. Distributed Application: Communicates with other applications using APIs.
  4. AI Application: Bots used to communicate with users.
  5. IoT Application: Internet of Things applications, such as smart watch sensors.
  6. Mobile Application: Native applications for mobile devices, e.g., camera, calculator.
  7. 2D and 3D Gaming Applications: Applications for gaming consoles, e.g., Xbox.

What is a Network?

A computer network comprises a group of computers connecting with each other to share information and devices.

Internet:

The Internet is a wide area network that connects computers all over the world. It includes:

  • LAN: Local Area Network
  • MAN: Metropolitan Area Network
  • WAN: Wide Area Network

Web:

In the early 1990s, Tim Berners-Lee introduced the concept of the web. Web applications can be:

  • Anonymous user: Users who access web apps without signing up or creating a user ID.
  • Authorized user: Users who log in and use a user ID to access the web app.

Tim Berners-Lee also introduced HTML (Hypertext Markup Language). HTML is a presentation language designed to:

  • Hypertext: Refers to content that links beyond what is immediately visible.
  • Markup: Refers to preparing content for presentation.

History of Markup Languages:

  • GML (Generalized Markup Language): The first presentation language.
  • SGML (Standard Generalized Markup Language): The second presentation language.
  • HTML (Hypertext Markup Language): The third presentation language.

Difference Between Presentation Language and Programming Language:

  • Presentation Language: Used for markup and presentation, but does not interact with users.
  • Programming Language: Can interact with users and perform logic and computations.

HTML Maintenance:

HTML is maintained by:

  • W3C (World Wide Web Consortium)
  • WHATWG (Web Hypertext Application Technology Work Group)

Types of Servers in Software Development

Types of Servers:

  1. Sandbox: Repository for libraries.
  2. Development: Used during the development process.
  3. Quality: Used to check or test the product.
  4. Production: Used for the client’s final product.

3-Tier Architecture:

  1. UI Layer: Builds the user interface (presentation).
  2. Business Layer: Contains the logic for the program, deployed on the application server.
  3. Database Server: Manages data storage and retrieval, deployed on the database server.

Tier and Layers:

Tier: Refers to the physical infrastructure that runs and maintains the application (e.g., Data Tier for backups).

Layers: Refers to the logical architecture built by developers (e.g., Database Layer for creating databases, tables, and queries).

Full Stack Developer:

  • UI Layer: Technologies include HTML, CSS, Bootstrap, JavaScript, React, jQuery.
  • Business Layer: Technologies include Node.js, Express, API, Services.
  • Data Layer: Technologies include MongoDB, MySQL.

Web Server Types:

  1. Local Web Server: Available only on your machine. Examples include Tomcat, XAMPP, IIS.
  2. Cloud Server: Available on the cloud and accessible from anywhere. Examples include AWS, MongoDB, Azure.

What is a Website?

A website is a virtual directory that provides access to resources on a server:

  • Web: Refers to the location on a web server.
  • Site: Refers to the location on a drive and directory.

Difference Between Website and Web Application

Website:

A website generally consists of static resources that do not change frequently. It is primarily used to display information to users. Characteristics of websites include:

  • Static Content: Content is fixed and does not change dynamically based on user interaction.
  • Informational: Provides information such as articles, blogs, news, or corporate information.
  • Navigation: Typically involves straightforward navigation and limited user interaction.

Web Application:

A web application is more interactive and dynamic compared to a website. It often includes both static and dynamic resources. Characteristics of web applications include:

  • Dynamic Content: Content changes based on user input or interaction, such as user accounts, dashboards, or interactive forms.
  • Functionality: Provides complex functionality such as data processing, user authentication, and custom interactions.
  • User Interaction: Involves more sophisticated navigation and user interaction, including CRUD operations (Create, Read, Update, Delete).

Key Differences:

Here are the key differences between a website and a web application:

  1. Purpose: Websites are designed to provide information, while web applications are designed to offer interactive functionalities and user-specific services.
  2. Content: Websites primarily feature static content; web applications feature both static and dynamic content.
  3. User Interaction: Websites have limited user interaction, while web applications require user input and provide interactive features.
  4. Complexity: Websites tend to be simpler, while web applications are often more complex with backend services and databases.

Create Project

Step 1: Create a Folder

Create a folder in the C drive or D drive with the desired project name.

Step 2: Open the Folder in an IDE

Open the newly created folder in Visual Studio Code (VSCode) or any other Integrated Development Environment (IDE).

Step 3: Initialize the Project

Run the following command in the command prompt to initialize your project:

npm init -y

This command is explained as follows:

  • npm: Node Package Manager
  • init: Initializes the project
  • -y: Accepts all default settings for project metadata

This will generate a new file package.json, which contains project meta-data. The file is formatted using JSON (JavaScript Object Notation).

Step 4: Add Folders and Files

Add the following folders to your project:

  • public: A repository for static resources such as .html, .txt, .pdf, .images, etc.
  • src: A repository for dynamic resources such as .css, .scss, .js, .ts, etc.

Add an index.html file to the public folder. Note that every project starts with an index.html file as it serves as the startup page for the web application.

What is a Web Page?

A web page is a hypertext document that provides a user interface for interacting with the resources in an application. Web pages are classified into two types:

  • Static Page: Consists of fixed content and does not change dynamically. Files include .html, .htm.
  • Dynamic Page: Can generate content dynamically based on user interaction or other factors. Files include .jsp, .php, etc.

HTML [Hypertext Markup Language]

What is Hypertext?

The term “hyper” means beyond. It specifies that text contains information beyond what you see.

What is a Markup Language?

The term markup is derived from “Marking up”, which prepares content for presentation. It is used to present content in a browser.

Evolution of HTML

The evolution of HTML has progressed through several stages:

  1. GML [Generic Markup Language]: The first markup language used on the Internet, developed by CERN (Council for European Research and Nuclear Labs).
  2. SGML [Standard Generic Markup Language]
  3. 1990: Tim Berners-Lee introduced HTML.
  4. IETF [Internet Engineering Task Force]: Developed HTML up to version 3.2.
  5. 2004: WHATWG and W3C started the evolution of HTML to version 4.0.
  6. W3C [World Wide Web Consortium] and WHATWG [Web HyperText Application Technology Work Group] continue to evolve HTML.
  7. HTML 5: The latest version of HTML.

HTML Rendering Process

The HTML rendering process involves several steps:

  • Bytes → Characters → Tokens → Nodes → DOM → Rendering → Layout → Paint

HTML Elements Classification

HTML elements are classified into the following types:

1. Normal Elements:

  • Returns a presentation directly on callback.
  • Does not stop the presentation.
  • Does not require any additional attributes.
  • Requires a start and end tag.
  • Example: <b>start</b>end

2. Void Elements:

  • The term void indicates “no-return type or value”.
  • Void discards the return value.
  • Void element can’t return any presentation directly on callback.
  • Requires additional attributes.
  • Returns and stops the presentation implicitly.
  • Does not require an “end” tag.
  • Often known as “self-ending element”.
  • Example: <img/>

3. RC Data Elements:

  • Rich Content Data Elements will not allow any other element within the context.
  • Used only for plain text.
  • Example: <textarea>Not allow any tag</textarea>

4. Raw Text Elements:

  • Elements that are presented using raw text.
  • No tags required.
  • Raw text elements are represented using the & symbol.
  • Example: &copy, &#8377

5. Foreign Elements:

  • Elements used in HTML but are not native to HTML and need to be imported from a library.
  • Examples: Canvas, SVG, MathML

HTML File Extension and Structure

HTML File Extension

  • .html
  • .htm

Both extensions are used for HTML files and function the same way. Some older software systems, like MS Word, limited file name lengths to 8 characters for names and 3 characters for extensions, leading to the use of both .html and .htm.

Structure of HTML Page

The HTML structure contains two main parts:

  1. Document Declaration:

    This tells the browser which HTML version is being used. It is not a tag but a declaration comment for the browser.

    Syntax: <!DOCTYPE html> (for HTML 5 version)

  2. Document Scope:

    Defines the start and end of an HTML document.

    Syntax: <html></html>

    Document Attribute: lang="value" specifies the language used for the document. Example values include en-in, en-us, en-gd.

    Within the Document Scope, there are two main sections:

    1. Head Section:

      Contains information loaded into the browser memory and accessed as needed. It is cleared from memory when the browser is closed.

      Syntax: <head></head>

      Typical contents include:

      • Title: Defines the title displayed in the browser's title bar and for bookmarking the page.
      • Link: Links to external resources like stylesheets.
      • Meta: Provides metadata such as character set, description, and keywords.
      • Script: Includes JavaScript code.
      • Style: Contains CSS styles.

      Example:

      <head>
        <title>Index Page</title>
    </head>
    2. Body Section:

      Contains the main content of the HTML document, which is visible to the user.
